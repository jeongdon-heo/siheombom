# 시험봄(siheombom) 프로젝트 요구서

## 프로젝트 개요
PDF 시험지와 정답지를 업로드하면 AI가 문항을 분석하여 온라인 단원평가를 만들어주는 교사용 웹앱.
- 프로젝트명: siheombom (시험봄)
- NugaBom(누가봄) 시리즈의 두 번째 앱
- 배포: Vercel (siheombom.vercel.app)

## 기술 스택
- Vite + React (프론트엔드)
- Supabase (Auth, Database, Storage)
- Anthropic Claude API + Google Gemini API (선택 가능)
- pdf.js (PDF → 이미지 변환)
- Tailwind CSS (스타일링)

---

## 사용자 유형 및 인증

### 1. 역할 선택 화면
앱 첫 화면에서 "교사" / "학생" 역할을 선택한다.

### 2. 교사 인증
- **회원가입**: 이름, 학급명, 이메일, 비밀번호 → 가입 시 학급코드(6자리 영숫자) 자동 생성
- **로그인**: 이메일 + 비밀번호 (Supabase Auth 사용)
- 각 교사의 데이터(시험, 학생, 결과)는 완전히 독립적으로 관리
- 동료 교사에게 보급할 수 있도록 멀티 교사 지원

### 3. 학생 접속 (학급코드 방식)
- 학생은 회원가입 없음
- **학급코드 입력** → 해당 교사의 학급으로 연결 (학급명, 교사명 표시되어 확인 가능)
- **이름 + 번호 입력** → 시험 목록으로 이동
- 초등학생(4학년)이 사용하므로 최대한 간단하게

## 교사 기능

### 홈 화면
- 학급코드를 크게 표시 + 복사 버튼 (칠판에 적어줄 용도)
- 설정, 시험 만들기, 시험 관리, 결과 분석 메뉴

### 설정
- **AI 서비스 선택**: Gemini / Claude 토글 선택
- **API 키 입력**: 선택한 서비스에 맞는 API 키 입력란 + 연결 테스트 버튼
  - Gemini: Google AI Studio에서 발급 (AIza...)
  - Claude: Anthropic Console에서 발급 (sk-ant-...)
- 학급코드 변경 (재생성) 기능
- 학급명 변경

### API 호출 방식
- **Claude API**: `https://api.anthropic.com/v1/messages` (model: claude-sonnet-4-20250514)
- **Gemini API**: `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key={API_KEY}`
- API 키는 웹에서 직접 입력하는 방식 (서버 환경변수 아님). 동료교사가 자기 키를 넣어서 사용하기 위함.

### 시험 만들기 (핵심 기능)
3단계 프로세스:

**1단계: PDF 업로드**
- 과목, 단원 입력
- 시험지 PDF 업로드 → pdf.js로 페이지별 이미지 변환
- 정답지 PDF 업로드 → 동일하게 변환

**2단계: AI 분석**
- 시험지 이미지 + 정답지 이미지를 AI에게 전송
- AI가 각 문항을 분석하여 JSON으로 반환:
  ```json
  {
    "number": 1,
    "text": "문제 텍스트",
    "type": "multiple_choice | short_answer | essay",
    "options": ["①보기1", "②보기2", "③보기3", "④보기4"],
    "correct_answer": "정답",
    "points": 5,
    "page": 1,
    "position": "top | middle | bottom",
    "learning_objective": "관련 학습 목표"
  }
  ```
- 각 문항의 위치(page, position)를 기반으로 시험지 이미지에서 해당 영역을 crop하여 문항별 이미지 생성

**3단계: 교사 검수/수정**
- AI가 추출한 문항 목록을 보여줌 (아코디언 방식)
- 각 문항별로 수정 가능: 문제 텍스트, 유형, 보기, 정답, 배점, 학습 목표
- 이미지 영역 조정: 상단/중단/하단 선택하면 다시 crop
- 문항 추가/삭제 가능
- 검수 완료 후 시험 저장

### 시험 관리
- 시험 목록 표시 (과목, 단원, 문항수, 응시인원, 생성일)
- 시험 삭제 기능

### 결과 분석
- 시험별 응시 학생 목록 (번호순 정렬)
- 응시 인원, 평균 점수 표시
- 학생 클릭 → 해당 학생의 누적 기록 조회

### 학생별 누적 기록
- 점수 추이 바 차트 (시험별 점수율)
- 각 시험의 상세 결과
- AI 분석 생성 버튼: 시험 결과를 기반으로 잘한 점, 부족한 점을 한 문단(3~5문장)으로 생성
  - 초등학교 교사가 학부모에게 전달하는 따뜻하고 구체적인 톤
  - 구체적 문항 언급, 학습 방향 제안 포함

## 학생 기능

### 학급코드 입력 화면
- 큰 입력칸에 학급코드 입력 (대문자 자동 변환)
- 코드 검증 → 해당 교사의 학급명, 교사명 표시

### 이름/번호 입력 화면
- 상단에 "4학년 2반 · 김선생 선생님" 같이 학급 정보 표시
- 이름, 번호 입력 → 입장

### 시험 선택 화면
- 상단에 "3번 홍길동" 같이 학생 배지 표시
- **이어서 풀기**: 미제출 시험이 있으면 진행률 바와 함께 최상단에 표시 → 탭하면 마지막 상태 그대로 이어보기
- **시험 목록**: 응시 가능한 시험 카드 (과목명 첫 글자 아이콘, 과목, 단원, 문항수)
- **완료된 시험**: 흐리게 표시 + 점수 표시

### 시험 보기 화면 (가장 중요한 화면)
- **상단**: 학생 이름 + 과목·단원 + 진행 카운터
- **진행 바**: 전체 문항 대비 작성 완료 비율
- **문항 번호 점(dot)**: 번호를 탭하면 해당 문항으로 이동. 작성 완료=초록, 현재=보라, 미작성=회색
- **문제 영역**:
  - 문항 번호 + 배점
  - **문항 이미지** (시험지에서 crop한 이미지) — 학생이 그림을 보고 풀어야 하는 문제가 있으므로 반드시 보여줘야 함
  - 문제 텍스트
  - 답 입력 UI (유형별로 다르게):
    - **객관식**: 보기 카드 버튼 (선택하면 강조)
    - **단답형**: 큰 텍스트 입력칸 (가운데 정렬)
    - **서술형**: 텍스트에어리어
- **하단 네비게이션**: ◀이전 / 현재번호 / 다음▶ | 마지막 문항에서는 "제출 📮" 버튼
- **제출 확인 모달**: 작성/미작성 문항 수 표시, 미작성 시 경고

### ⭐ 시험 이어보기 (핵심 요구사항)
- 학생이 시험 중 **뒤로가기, 브라우저 닫힘, 앱 튕김** 등으로 이탈해도 답안이 보존되어야 함
- 매 답안 변경 시마다 자동 저장 (DB 또는 로컬 상태 업데이트)
- 같은 이름+번호로 다시 입장하면 미완료 시험이 "이어서 풀기"로 표시됨
- 이어보기 시 마지막 문항 위치 + 기존 답안 모두 복원

### 시험 결과 화면
- **원형 점수 그래프** (SVG로 점수율을 원형 프로그레스로 표시)
- 학생 이름, 점수/총점, 정답수
- **문항별 정오 표시**: 정답=초록, 오답=빨강, 서술형=노랑
  - 각 문항: 내 답 + 정답(오답인 경우) + 득점
- **AI 분석 ("선생님의 한 마디")**: 자동 생성 or 버튼으로 생성
  - 잘한 점과 부족한 점을 따뜻한 톤으로 한 문단 제시

## 데이터 구조 (Supabase)

```sql
-- 교사 (Supabase Auth + profiles 테이블)
teachers: id, name, email, class_name, class_code, provider, api_key_encrypted, created_at

-- 학생 (교사별)
students: id, teacher_id, name, number, created_at

-- 시험
exams: id, teacher_id, subject, unit, created_at

-- 문항
questions: id, exam_id, number, text, type(multiple_choice/short_answer/essay), 
           options(jsonb), correct_answer, points, image_url, 
           learning_objective, page, position

-- 시험 세션 (학생의 시험 응시 기록)
exam_sessions: id, exam_id, student_id, answers(jsonb), current_index,
               started_at, submitted_at, submitted(boolean),
               score, max_score, results(jsonb), analysis(text)
```

## 채점 로직

- **객관식**: 학생 답안과 정답을 비교. 번호 기호(①②③④⑤) 제거 후 비교도 수행
- **단답형**: 정확 일치 + 공백 제거 후 비교. 정답을 여러 형태로 등록 가능하면 더 좋음 (예: "3/4", "4분의 3")
- **서술형**: 자동 채점 불가 → isCorrect: null로 처리. 교사가 수동 채점할 수 있으면 좋음

## 개발 우선순위

1. Supabase 세팅 + 교사 회원가입/로그인
2. 학급코드 시스템 + 학생 접속
3. PDF 업로드 → AI 문항 분석 → 교사 검수
4. 학생 시험 보기 화면 + 자동 저장 + 이어보기
5. 자동 채점 + 결과 화면
6. AI 분석 코멘트 생성
7. 학생별 누적 기록 + 차트

## 참고
- 동료교사 보급용이므로 각 교사가 자기 API 키를 웹에서 입력하여 사용
- API 키를 서버 환경변수가 아닌 DB에 저장 (암호화 권장)
- 학생은 별도 비밀번호 없이 학급코드 + 이름 + 번호로만 접속
